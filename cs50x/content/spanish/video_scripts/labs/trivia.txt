En este laboratorio, su tarea será crear un sitio web que haga preguntas triviales a los usuarios y, en función de las respuestas de los usuarios, brinde alguna retroalimentación sobre si la respuesta del usuario era correcta o incorrecta. Aquí podrá ver cómo se verá su producto final. No es necesario que elija estas preguntas exactas, pero deberá hacer una pregunta de opción múltiple con múltiples opciones diferentes, una de las cuales es la correcta, así como una pregunta de respuesta libre, donde el usuario puede escribir su propia respuesta y luego verificar si esa respuesta es correcta.

Sumergámonos en el código de distribución que le proporcionamos como punto de partida para trabajar en este laboratorio. Dentro de index.html hay una página HTML que actuará como punto de partida para este sitio web. Notará que en la parte superior hemos dejado un espacio para que pueda agregar algo de JavaScript, más sobre eso en un momento, pero el JavaScript es donde, en última instancia, proporcionará retroalimentación al usuario, agregando cierta lógica para manejar casos en los que el usuario responde correctamente la pregunta, así como la lógica para manejar casos en los que el usuario responde incorrectamente también.

Dentro del cuerpo de la página, mostramos un gran título en la parte superior que dice "Trivia!" Y luego tenemos un título más pequeño que dice "Parte 1: Opción múltiple". Aquí es donde agregará su pregunta de opción múltiple, preguntando la pregunta y luego también agregando botones donde el usuario puede hacer clic en un botón para indicar cuál de las opciones de opción múltiple les gustaría seleccionar. Abajo está la Parte 2, respuesta libre, donde incluirá su pregunta de respuesta libre, así como un campo de texto, donde el usuario puede escribir su respuesta a su pregunta y un botón donde pueden enviar su respuesta.

Para ejecutar su sitio web, en su terminal cuando esté dentro del directorio del laboratorio, puede ejecutar http-server para iniciar un servidor HTTP. Esto le dará una URL para visitar en el puerto 8080, y si hago clic en esa URL y en Abrir, podré elegir index.html, la página que quiero visitar, y aquí verá el esquema que le damos para su sitio web de trivia. Actualmente no hay nada debajo de la Parte 1 o Parte 2, pero ahí es donde entra usted, al proporcionar la pregunta y agregar lógica para manejar la respuesta del usuario.

Entonces, ¿qué vas a hacer exactamente en este laboratorio? En la Parte 1 del laboratorio, comenzará agregando una pregunta de opción múltiple a la página HTML. Utilizará un encabezado h3 para el texto de la pregunta, recordando que hay h1, 2, 3, 4, 5 y 6 para diferentes niveles de encabezados. Ya hemos usado h1 y h2, por lo que h3 actuará como el encabezado aquí. Y luego agregarás un botón para cada opción de respuesta posible. Probablemente deberías tener al menos tres opciones de respuesta diferentes de las cuales exactamente una de ellas debería ser la respuesta correcta.

A partir de ahí, querrás escribir algún JavaScript para agregar algo de lógica a esta parte de la aplicación. Cuando se hace clic en una opción de respuesta incorrecta, cualquier botón que sea la respuesta incorrecta, el botón debería volverse rojo y debajo de la pregunta debería aparecer un texto que diga "incorrecto". Entonces, el botón se vuelve rojo, aparece un texto para informar al usuario que respondió incorrectamente su pregunta. Mientras tanto, cuando se hace clic en la respuesta correcta, entonces el botón debería volverse verde y debajo de esa pregunta debería aparecer un texto que diga que el usuario respondió correctamente.

Mientras tanto, en la Parte 2, harás algo muy similar, excepto que en lugar de hacer esto para una pregunta de opción múltiple, lo harás para una pregunta de respuesta libre. Entonces comenzarás eligiendo una pregunta de respuesta libre y agregando esa pregunta a la página HTML nuevamente, usando un encabezado h3 para el texto de la pregunta. Pero esta vez, en lugar de un botón para cada posible respuesta, utilizarás una entrada para la respuesta y un botón para confirmar la respuesta una vez que el usuario haya escrito su respuesta.

Nuevamente, usarás algún JavaScript para agregar algo de lógica a este sitio web también. Si el usuario escribe una respuesta incorrecta, el propio campo de texto debería volverse rojo y debajo de la pregunta debería aparecer un texto que diga "incorrecto". Por el contrario, cuando se escribe la respuesta correcta y el usuario presiona el botón, entonces el campo de texto debería volverse verde y debajo de la pregunta debería aparecer un texto que diga que el usuario respondió correctamente.

Al intentar implementar esta lógica, hay algunas técnicas de JavaScript que probablemente le resulten útiles. El primero es usar document.querySelector, recuerde que puede usar document.querySelector para seleccionar algún elemento de la página por su nombre de etiqueta, por su ID o por su clase o desde algún otro selector CSS y puede usar eso para que dentro de su código JavaScript pueda acceder a un botón, por ejemplo, o a un campo de entrada para acceder a lo que escribió el usuario o hacer algo con ese botón, como cambiar su estilo o cambiar lo que hay dentro del HTML de algún otro párrafo que existe en su página.

Probablemente también querrá usar la función addEventListener, que puede agregar algún código que puede responder a algún evento que sucede en su página web. Por ejemplo, podría agregar un escucha de evento a un botón tal que cuando se hace clic en un botón, se va a ejecutar otra función. Esa función podría cambiar el estilo del botón para que tenga un color rojo o un color verde o podría cambiar el HTML de alguna parte de su página para mostrar algún texto, como correcto o incorrecto o incluso podría hacer una combinación de ambos. La función addEventListener es una herramienta muy poderosa en JavaScript para permitir que tenga código JavaScript que pueda responder a eventos que están sucediendo en su página.
