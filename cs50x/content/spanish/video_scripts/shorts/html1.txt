Pero hablemos de un lenguaje que es realmente fundamental para cada página web, que es HTML. HTML es el Lenguaje de Marcado de Hipertexto. HTML es un lenguaje, pero no es un lenguaje de programación.

HTML no tiene variables. No tiene lógica ni funciones ni nada por el estilo. No podemos hacer ninguna programación en HTML per se. A veces escucharás a la gente describirse a sí mismos como programadores de HTML, lo cual no es del todo exacto. No podemos escribir programas de HTML.

HTML solo se usa para marcar texto. Se llama lenguaje de marcado. Y lo que hace este marcado es que utilizamos etiquetas en HTML y estas etiquetas, este marcado, define semánticamente la estructura de una página y hace que el texto plano que existe entre las etiquetas se interprete de diferentes maneras en los navegadores. Y quizás sea mejor explicarlo mediante una ilustración.

Aquí hay una página HTML muy simple, no un programa en HTML, nuevamente, una página HTML. Y sabemos que es una página HTML porque hemos englobado todo con etiquetas HTML. Entonces, esto es lo que parece una etiqueta HTML. Está entre corchetes angulares. Y observe en la parte superior que tenemos HTML y en la parte inferior después de haber hecho aparentemente mucho más HTML, tenemos corchetes angulares con una barra, HTML. Entonces, eso es una especie de límite entre lo que es HTML y lo que no lo es. Y, por supuesto, convencionalmente, así como escribiste todos tus programas en C con extensiones .C, todos tus archivos HTML terminarán con extensiones .HTML. Pero hay más cosas sucediendo aquí. No solo tenemos estas etiquetas HTML. Aparentemente, tenemos algo llamado una etiqueta de encabezado. Bueno, ¿qué es eso?

Bueno, tal vez sea mejor distinguir por medio de un cuerpo, siendo el contenido de la página web. Entonces, tal vez la etiqueta de encabezado define cosas que no están dentro de la ventana del navegador en sí, pero que son importantes de alguna manera para que nuestra página web se reproduzca correctamente. Por ejemplo, dentro de la etiqueta de encabezado tenemos etiquetas de título.

Entonces, el título es "hola mundo", eso será lo que se muestre en la pestaña en Chrome o en safari o Firefox-- cualquier navegador que prefiera-- eso es lo que aparecerá en el título. Y antes de las pestañas, aparecería en toda la ventana del navegador y solo podía tener una página abierta en una ventana del navegador a la vez. Entonces ese será el título de mi página en la pestaña o en la barra de la ventana del navegador, "hola mundo". Y luego el contenido de mi página web será "mundo, hola". Así que echemos un vistazo a cómo podría lucir algo como esto. Esta es una página HTML muy simple. 

Así que estoy aquí en mi CS50 IDE y he ampliado un poco. Y solo voy a abrir hola. HTML y mostrarles que esto es prácticamente el contenido de la página que vimos antes. Mi página HTML simple, etiquetas de encabezado, etiquetas de título, cuerpo, etc. He tabulado esto para que sea limpio. 

Luego, lo que puedo hacer en mi IDE es simplemente previsualizar la página. Y ahí está. El contenido de mi página es "mundo, hola", y no veo nada de las etiquetas de encabezado allí. Es solo el contenido del cuerpo. "Mundo. hola". Y otra vez, el cuerpo solo dice "mundo, hola". Falta la otra parte.

Entonces, eso es todo lo que es. Esta es una página HTML básica y simple. Ahora he indentado mi HTML para que sea muy agradable y organizado, pero en realidad no tengo que hacerlo. Podría hacer que se vea bastante mal. Y esto seguiría funcionando. Esta sería la misma página web exacta. Solo he eliminado todo el espacio en blanco.

Resulta que el espacio en blanco es datos. Y cuando enviamos datos del remitente al receptor, del servidor al cliente, los datos cuestan dinero. Y, por lo tanto, eliminar el espacio en blanco es una buena idea si eres alguien que proporciona mucho contenido web. Es una mala idea si eres alguien que está aprendiendo esto y quieres tenerlo bien organizado. Esto es mucho más fácil de analizar que esto. Pero es funcionalmente idéntico.

La sangría y cosas así en realidad no importan en HTML. Todo lo que importa son las etiquetas de apertura y cierre en el orden correcto. Pero observe lo que sucedió aquí. La marca nos da una manera de comunicar información adicional sobre lo que hemos escrito. La parte "hola, mundo" se interpretó como el título. Y la parte "mundo, hola" se interpretó como el contenido o lo que debería ser visible en mi página web.
