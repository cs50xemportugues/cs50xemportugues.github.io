Así que las variables tienen dos grandes diferencias con respecto a C. Ya no tenemos que especificar un tipo, lo cual es bastante interesante. Y podemos declararlas solo mediante la inicialización. Tal vez recuerde en C que podíamos declarar una variable diciendo, por ejemplo, int x punto y coma, pero sin asignarle un valor, sin inicializarla. En Python, solo podemos declarar variables inicializándolas. Entonces, donde en C podríamos decir algo así como int x igual a 54 punto y coma. En Python, solo decimos x igual a 54 y eso nos crea una nueva variable en Python llamada x y asigna el valor 54. Y observe aquí, las instrucciones de Python no necesitan terminar con punto y coma. Entonces, eso podría ser bueno si usted es el tipo de persona que, como yo, a menudo olvida poner un punto y coma al final de una línea. En Python no es necesario incluirlos. Puede incluirlos y no tendrá ningún problema, pero también puede omitirlos para que su código se vea un poco más limpio. De manera similar, podemos declarar lo siguiente, cadena de texto frase igual a esto es CS50 y para hacer esto en C tendríamos que incluir la biblioteca CS50 porque string no es un tipo de datos nativo en C. Pero sí lo es en Python. Solo podemos decir, frase igual a esto es CS50. De hecho, ni siquiera tenemos que usar comillas dobles. Python admite cadenas tanto con comillas dobles como con comillas simples. Y esto es realmente útil si necesita declarar una cadena que contenga comillas, simplemente alterne entre el uso de comillas simples en el exterior, comillas dobles en el interior, comillas simples dentro de eso, y así sucesivamente, lo cual es realmente útil si es el tipo de persona que trabaja con mucho texto, por ejemplo, particularmente con bases de datos.
