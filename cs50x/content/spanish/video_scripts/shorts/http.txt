Hablemos sobre otro protocolo - el Protocolo de Transferencia de Hipertexto o HTTP. Ya hemos hablado de IP y TCP en videos anteriores, los cuales son protocolos que dictan cómo se mueve la información de una máquina a otra, de un programa a otro o de un servicio a otro a través de la red de routers y máquinas que forma internet.

Pero por lo general, eso no es todo, ¿verdad? Por lo general, cuando enviamos información, el programa en sí mismo, cuando se recibe datos, por ejemplo, en un correo electrónico a través del puerto TCP 25 o una solicitud de página web a través del puerto 80, por lo general hay un sistema de reglas para procesar lo que hemos recibido. HTTP es un ejemplo de tal protocolo.

HTTP es el único protocolo de capa de aplicación del que hablaremos. Es otro conjunto de reglas que dictan cómo se transmitirá y procesará la información a través de internet. En particular, HTTP especifica exactamente cómo se debe hacer una solicitud de una página web y cómo un servidor, una máquina que aloja páginas web, entrega esa información a los clientes.

Entonces, este protocolo realmente no tiene nada que ver con cómo se mueve la información de un punto A a un punto B. Es realmente el sistema de reglas para - básicamente, es las reglas de compromiso para trabajar con una página web, similar a cuando alguien te saluda con la mano, se espera que respondas agitando la mano. Eso es algo así como un protocolo humano convencional. El protocolo HTTP solo dice, si desea solicitar una página web, asegúrese de que su formato sea así, algo así como el formato de una carta comercial, por ejemplo. Y la respuesta también vendrá de la misma manera. Hay otros protocolos de capa de aplicación de los que no hablaremos en videos. Pero estos incluyen cosas como el Protocolo de Transferencia de Archivos, el Protocolo Simple de Transferencia de Correo para enviar correos electrónicos, el Servicio de Distribución de Datos, el Protocolo de Escritorio Remoto, RDP, que se usa si desea acceder a su computadora de forma remota desde otra computadora, XMPP, que se conoce comúnmente como Jabber o chat, y este es el protocolo para usar servicios de chat. Y hay muchos, muchos otros.

Entonces, cada vez que esté utilizando un servicio, el servicio espera que la información se reciba, que se reciba una solicitud en un formato específico y que se requiera que devuelva información en un formato muy específico también.
