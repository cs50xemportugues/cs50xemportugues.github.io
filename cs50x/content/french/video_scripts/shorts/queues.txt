Donc, si vous avez regardé la vidéo sur la pile, cela risque de vous sembler un peu comme du déjà-vu. C'est un concept très similaire, mais avec une petite variation. Nous allons parler maintenant des files. Une file, similaire à une pile, est un autre type de structure de données que nous pouvons utiliser pour maintenir des données de manière organisée. Tout comme une pile, elle peut être mise en œuvre comme un tableau ou une liste chaînée. Contrairement à une pile, les règles que nous utilisons pour déterminer quand les éléments sont ajoutés et supprimés d'une file sont un peu différentes. Contrairement à une pile, qui est une structure LIFO, dernier entré, premier sorti, une file est une structure FIFO, premier entré, premier sorti. Vous avez probablement une analogie avec les files. Si vous avez déjà été en file d'attente dans un parc d'attractions ou dans une banque, il y a une sorte de structure de mise en œuvre d'équité. La première personne en file d'attente à la banque est la première personne à qui le caissier parle.

Ce serait une sorte de course vers le bas si la seule façon de parler au caissier était d'être la dernière personne en file d'attente. Tout le monde voudrait être le dernier en file d'attente, et la personne qui est arrivée en premier et a attendu depuis un moment pourrait y être pendant des heures et des heures avant d'avoir la chance de retirer de l'argent à la banque.
