Parlons d'un autre protocole – le Hypertext Transfer Protocol ou HTTP. Nous avons parlé d'IP et de TCP dans les vidéos précédentes. Et ce sont des protocoles qui dictent comment les informations se déplacent d'une machine à une autre, d'un programme à un autre ou d'un service à un autre via Internet, via ce réseau de routeurs et de machines. 

Mais ce n'est généralement pas tout, n'est-ce pas ? Normalement, lorsque nous envoyons des informations, le programme lui-même – lorsque les données sont reçues, disons, par exemple, dans le courrier électronique via le port TCP 25 ou une demande de page Web via le port 80, il y a généralement un système de règles pour traiter ce que je viens de recevoir. Et HTTP est un exemple de ce type de protocole. 

HTTP est le seul protocole de couche application dont nous allons parler. Mais c'est un autre ensemble de règles dictant comment l'information doit être transmise et traitée via Internet. Plus particulièrement, HTTP spécifie exactement comment faire une demande de page Web et exactement comment un serveur, une machine hébergeant des pages Web, renvoie cette information aux clients. 

Donc, ce protocole n'a rien à voir avec la façon dont les informations se déplacent de A à B. C'est vraiment le système de règles pour – c'est essentiellement les règles d'engagement pour travailler avec une page Web, similaire à quand quelqu'un vous fait signe de la main, il faut répondre en faisant signe. C'est en quelque sorte un protocole conventionnel humain. Le protocole HTTP se contente de dire : si vous voulez demander une page Web, assurez-vous que votre format ressemble à ceci – un peu comme la mise en forme d'une lettre commerciale, par exemple. Et la réponse viendra également selon ce protocole. Il y a d'autres protocoles de couche application dont nous ne parlerons pas dans les vidéos. Mais ceux-ci comprennent des choses comme le protocole de transfert de fichiers, le protocole de transfert de courrier simple pour l'envoi de courriels, le service de distribution de données, le protocole de Bureau à Distance (RDP) qui est utilisé si vous voulez accéder à distance à votre ordinateur à partir d'un autre ordinateur, XMPP, qui est souvent connu sous le nom de Jabber ou de chat. C'est donc le protocole pour utiliser les services de chat. Et il y en a beaucoup, beaucoup d'autres. 

Ainsi, chaque fois que vous utilisez un service, le service attend que l'information soit reçue – une demande de recevoir– dans un format très particulier et doit également retour
