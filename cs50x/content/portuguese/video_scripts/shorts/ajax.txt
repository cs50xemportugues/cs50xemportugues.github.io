Enquanto você continua explorando o JavaScript, há uma técnica que pode ser muito útil ao construir sites mais complexos, chamada de Ajax. Até agora, em nossa exploração do JavaScript, só fomos capazes de fazer coisas simples para fins de ilustração, como clicar em um botão e mudar a cor do site ou algumas palavras. Mas tudo o que fizemos até agora foi realizado lado do cliente, ou seja, localmente em nosso computador, sem sair deste ambiente. No entanto, podemos fazer requisições para o servidor e solicitar informações sem ter que recarregar a página inteira. Ajax permite atualizar seções de uma página de modo a criar uma experiência mais agradável para o usuário. Antigamente, Ajax significava algo como JavaScript Assíncrono em XML, mas agora se utiliza principalmente JSON em vez de XML. Mesmo assim, Ajax se tornou o nome técnico desse processo. As técnicas que se podem usar com Ajax são ilimitadas. É possível, por exemplo, atualizar as pontuações dos jogos em um site esportivo ou notificações de e-mail na caixa de entrada sem ter que recarregar a página inteira. Mas, usualmente, o processo fica ocorrendo em loop. Para exemplificar, o curso mostrará como realizar uma solicitação para o servidor quando o usuário clica em um botão e algo acontece. No entanto, essa interação não ficará localizada apenas na máquina do usuário. Para realizar isso, é necessário criar um objeto JavaScript especial chamado XMLHttpRequest. Esse objeto permite realizar solicitações assíncronas para um servidor para obter mais informações, e é muito fácil de criar. Uma vez criado, é preciso declarar um comportamento onreadystatechange, que nada mais é do que um jeito detalhado de descrever as etapas de um processo que ocorre em um site ao carregar ou executar uma ação. Em seguida, é preciso definir um estado usando um função anônima que descreverá o que o site deverá fazer quando o estado mudar. Existem cinco estados possíveis de mudança. O estado 0 significa que a solicitação ainda não foi iniciada, e o estado 4, que é o objetivo final, significa que todo o conteúdo da página foi carregado. Uma vez feito isso, é necessário enviar a solicitação e a página será atualizada. Há uma forma um pouco diferente de fazer isso usando o jQuery, algo que é mais comum hoje em dia, mas o curso apresenta explicitamente a versão em JavaScript puro para ilustração. No site, há um exemplo de como realizar uma solicitação para o servidor para alterar a página de modo assíncrono usando Ajax. É possível atualizar informações sem ter que atualizar toda a página. É uma técnica interessante porque possibilita enviar informações para o servidor não só localmente, mas para outros servidores também. Assim, é possível melhorar a experiência do usuário.
