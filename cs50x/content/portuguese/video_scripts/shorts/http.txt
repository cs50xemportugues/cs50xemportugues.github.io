Vamos falar sobre mais um protocolo: o Protocolo de Transferência de Hipertexto, ou HTTP. Nós já falamos sobre IP e TCP em vídeos anteriores. E esses são protocolos que ditam como a informação se move de máquina para máquina e de programa para programa ou serviço para serviço via internet, por meio dessa rede de roteadores e máquinas.

Mas isso geralmente não é toda a imagem, certo? Geralmente quando enviamos informações, o próprio programa, quando os dados são recebidos, por exemplo, em um e-mail via porta TCP 25 ou em uma solicitação de página da web via porta 80, geralmente há um sistema de regras lá para processar o que acabamos de receber. E o HTTP é um exemplo exato desse tipo de protocolo.

HTTP é o único protocolo de camada de aplicação que vamos falar. Mas é outro conjunto de regras que dita como a informação deve ser transmitida e processada via internet. Em particular, o HTTP especifica exatamente como alguém deve fazer uma solicitação para uma página da web e exatamente como um servidor, uma máquina que hospeda páginas da web, entrega essa informação de volta aos clientes.

Então, esse protocolo não tem nada a ver com como as informações se movem do ponto A ao ponto B. É realmente o sistema de regras para - é basicamente as regras do engajamento para trabalhar com uma página da web, semelhante a quando alguém acena a mão para você, você deve acenar de volta. Isso é uma espécie de protocolo humano convencional. O protocolo HTTP apenas diz que se você quiser solicitar uma página da web, certifique-se de que seu formato pareça com este - como o formato de um documento comercial, por exemplo. E a resposta também virá de acordo com este protocolo. Existem outros protocolos de camada de aplicação que não vamos falar em vídeos. Mas estes incluem coisas como o Protocolo de Transferência de Arquivos, Protocolo Simples de Transferência de E-mail para enviar e-mails, o Serviço de Distribuição de Dados, Protocolo de Área de Trabalho Remota, RDP, que é usado se você quiser acessar seu computador remotamente de outro computador, XMPP, que frequentemente é conhecido como Jabber ou chat, este é o protocolo para usar serviços de bate-papo. E há muitos, muitos outros.

Então, toda vez que você está usando um serviço, o serviço espera que as informações sejam recebidas - uma solicitação seja recebida - em um formato muito específico e é necessário que a informação seja devolvida em um formato muito específico também.
