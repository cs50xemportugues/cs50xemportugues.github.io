Neste laboratório, sua tarefa será criar um website que faça perguntas triviais aos usuários e, com base na resposta do usuário, fornecer feedback com base na resposta correta ou incorreta a essas perguntas. Aqui está o que o produto final pode parecer. Você não precisa escolher essas perguntas exatas, mas fará uma pergunta de escolha múltipla com várias respostas diferentes, das quais uma é correta, bem como uma pergunta de resposta gratuita, na qual o usuário pode digitar uma resposta própria e, em seguida, verificar se essa resposta está correta.

Vamos mergulhar no código de distribuição que lhe damos como ponto de partida para trabalhar neste laboratório. Dentro do index.html há uma página HTML que vai agir como ponto de partida para este site. Você perceberá que lá em cima, deixamos um espaço para você adicionar algum JavaScript. Mais sobre isso em um momento, mas é no JavaScript que você vai fornecer feedback para o usuário, adicionando alguma lógica para lidar com casos em que o usuário responde corretamente à pergunta, bem como a lógica para lidar com casos em que o usuário responde incorretamente.

Dentro do corpo da página, exibimos um grande título no topo que diz Trivia! E depois temos um título menor que diz Parte 1: Escolha Múltipla. É aqui que você vai adicionar sua pergunta de escolha múltipla, perguntando algo e, em seguida, adicionando botões onde o usuário pode clicar em um botão para indicar qual das opções de escolha múltipla eles gostariam de selecionar. Abaixo está a Parte 2, resposta livre, onde você incluirá sua pergunta de resposta livre, bem como um campo de texto, onde o usuário pode digitar a resposta deles e um botão onde eles podem enviar sua resposta.

Para executar seu site, dentro do seu terminal quando você estiver no diretório do laboratório, você pode executar http-server para iniciar um servidor HTTP. Isso lhe dará uma URL para visitar na porta 8080, e se eu clicar nessa URL e depois em Abrir, poderei escolher index.html, a página que desejo visitar, e aqui você verá o esboço que lhe fornecemos para seu site de trivia. Agora, não há nada abaixo da Parte 1 ou Parte 2, mas é onde você entra, fornecendo a pergunta e adicionando lógica para lidar com a resposta do usuário.

Então, o que exatamente você vai fazer neste laboratório? Na Parte 1 do laboratório, você começará por adicionar uma pergunta de escolha múltipla na página HTML. Você usará um título h3 para o texto da pergunta, lembrando que há h1, 2, 3, 4, 5 e 6 para diferentes níveis de títulos. Já usamos h1 e h2, portanto, h3 atuará como o título aqui. E então você adicionará um botão para cada escolha possível de resposta. Você provavelmente deve ter pelo menos três opções de resposta diferentes que o usuário pode escolher, das quais exatamente uma delas deve ser a resposta correta.

A partir daí, você vai querer escrever algum JavaScript para adicionar alguma lógica a esta parte do aplicativo. Quando uma escolha de resposta incorreta é clicada, qualquer um daqueles botões que são a resposta errada, o botão deve ficar vermelho e o texto deve aparecer abaixo da pergunta, dizendo que está incorreto. Então, o botão fica vermelho, o texto aparece para informar ao usuário que ele respondeu incorretamente à sua pergunta. Enquanto isso, quando a opção de resposta correta é clicada, o botão deve ficar verde e o texto deve aparecer abaixo dessa pergunta dizendo que o usuário respondeu corretamente à pergunta.

Enquanto isso, na Parte 2, você fará algo muito semelhante, exceto que em vez de fazer isso para uma pergunta de escolha múltipla, você fará isso para uma pergunta de resposta gratuita. Então, você começará escolhendo uma pergunta de resposta gratuita e adicionando essa pergunta à página HTML novamente, usando um título h3 para o texto da pergunta. Mas desta vez, em vez de um botão para cada resposta possível, você usará uma entrada para a resposta e um botão para confirmar a resposta depois que o usuário digitou sua resposta.

Novamente, você usará algum JavaScript para adicionar alguma lógica a este site também. Se o usuário digitar uma resposta incorreta, o próprio campo de texto deve ficar vermelho e o texto deve aparecer abaixo da pergunta, dizendo que está incorreto. Por outro lado, quando a escolha correta de resposta for digitada e o usuário pressionar o botão, então o campo de texto deve ficar verde e o texto deve aparecer abaixo da pergunta dizendo que o usuário respondeu corretamente.

Ao tentar implementar esta lógica, existem algumas técnicas JavaScript que provavelmente serão úteis para você. O primeiro é usar document.querySelector, lembre-se de que você pode usar document.querySelector para selecionar algum elemento na página pelo nome da tag, pelo ID, ou pela classe ou de algum outro seletor CSS e você pode usar isso para que dentro do seu código JavaScript você possa acessando um botão, por exemplo, ou um campo de entrada para acessar o que o usuário digitou ou fazer algo com esse botão, como alterar seu estilo ou alterar o que está dentro do HTML de algum outro parágrafo existente em sua página.

Você também provavelmente desejará usar a função addEventListener, que é capaz de adicionar algum código que pode responder a algum evento que acontece em sua página da Web. Por exemplo, você pode adicionar um ouvinte de eventos a um botão de modo que, quando um botão for clicado, você executará alguma outra função. Essa função pode alterar o estilo do botão para que ele tenha uma cor vermelha ou uma cor verde ou ele pode mudar o HTML de parte de sua página para exibir algum texto, como Correto ou Incorreto ou até mesmo fazer uma combinação dos dois. AddEventListener é uma ferramenta muito poderosa em JavaScript que permite ter um código JavaScript que pode responder a eventos que estão realmente acontecendo em sua página.
