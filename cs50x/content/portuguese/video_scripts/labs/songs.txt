Neste laboratório, sua tarefa será escrever consultas em SQL que podem responder perguntas sobre um banco de dados de músicas. Como parte deste laboratório, forneceremos um banco de dados SQLite chamado songs.db, no qual há informações retiradas do Spotify sobre várias músicas diferentes e seus artistas. Em última análise, a primeira coisa que você vai querer fazer é entender a estrutura ou esquema das tabelas deste banco de dados.

Dentro de songs.db, você encontrará uma tabela chamada músicas que tem algumas colunas diferentes. Cada música tem um ID, que é um INTEGER, o nome da música, que é algum texto, o ID do artista, que é um INTEGER representando o artista da música – que retornaremos a esse assunto em um momento – e então outras informações sobre a música, como sua capacidade de dança, sua energia, sua chave, sua intensidade, bem como outras propriedades da música também.

Agora, vamos dar uma olhada no ID do artista. Para representar os diferentes artistas que as músicas podem ter, há outra tabela neste banco de dados chamada tabela de artistas. Cada artista tem um ID, que é um INTEGER, e um nome, que vai ser apenas algum texto. E agora a coluna ID do artista da tabela de músicas vai ser um INTEGER que corresponde ao ID de um desses artistas.

Então, vamos dar uma olhada em como esse banco de dados realmente é. Depois de ter baixado o laboratório, no terminal, posso digitar sqlite3 songs.db para abrir este banco de dados. E se eu quiser, posso digitar .schema para ter uma ideia do esquema deste banco de dados. Eu veria novamente, aqui está CREATE TABLE músicas, juntamente com todas as colunas para as músicas, assim como CREATE TABLE artistas, para ver todos esses artistas.

Então, se eu quisesse ver quais são as músicas, por exemplo, eu poderia escrever uma consulta como SELECT star FROM músicas, para selecionar todas as músicas. E eu veria todas as músicas, juntamente com todas as colunas que tenho para cada uma dessas linhas. Talvez eu não quisesse todas essas colunas. Então, também poderia filtrar meus resultados. Eu poderia escrever uma consulta como SELECT name FROM song para apenas selecionar a coluna de nome da minha tabela de músicas. Se eu fizer isso, só verei o nome de cada uma dessas músicas.

Em última análise, neste laboratório, vamos fazer oito perguntas diferentes sobre este banco de dados, e sua tarefa será escrever consultas para responder a cada uma dessas perguntas. Nós lhe fornecemos arquivos nos quais você pode colocar essas respostas. Por exemplo, o arquivo 1.sql é onde você incluirá sua resposta à primeira pergunta, escrevendo uma consulta SQL aqui que pode ser usada para responder a uma pergunta específica.

Quais são esses prompts? Bem, por exemplo, em 1.sql, você vai escrever uma consulta SQL para listar o nome de todas as músicas no banco de dados. E essa, na verdade, é uma consulta que acabei de mostrar para você – SELECT name FROM músicas. Então, dentro de 1.sql, você escreveria esta consulta, SELECT name FROM músicas, como uma consulta que você poderia executar para obter o nome de todas as músicas no banco de dados.

Em última análise, os outros prompts vão pedir outras informações sobre o banco de dados. Talvez não apenas olhando para o nome das músicas, mas talvez olhando para os artistas das músicas, olhando para o volume dessas músicas, olhando para sua capacidade de dança, talvez outros fatores também, filtrando e selecionando e juntando tabelas juntas conforme necessário, a fim de obter os resultados que você quer, para responder a todas essas perguntas apenas com base em um único banco de dados.
