Olá

Se você já começou a resolver a Lista de Problemas 1 no CS50 Lab, você pode continuar resolvendo o problema lá. Se você estiver começando a resolver este problema agora, use o CS50 IDE e siga as instruções abaixo!

Começando

O CS50 IDE é um "ambiente de desenvolvimento integrado" que nos permite programar "na nuvem", sem precisar instalar nenhum programa localmente. O CS50 nos oferece nossa própria "estação de trabalho" (incluindo espaço de armazenamento) dentro da qual nós podemos salvar nossos arquivos e pastas (também chamadas de "diretórios").

Acessando o CS50 IDE

Vá até a página ide.cs50.io e clique na opção "Log in" para acessar o CS50 IDE. Após clicar em "Log in", você será redirecionado para uma página em que você terá que acessar usando sua conta do GitHub (faça uma conta no GitHub caso você já não tenha uma). Após acessar usando sua conta do GitHub, o ambiente de desenvolvimento integrado (IDE) será carregado, e ele estará dividido em três partes. Próximo ao topo do CS50 IDE, estará o "editor de texto", que é a área onde você escreverá todos os seus programas. Próximo à parte inferior do CS50 IDE, estará uma "janela do terminal" (de cor azul clara, por padrão), que é o que chamamos de interface de linha de comando, ferramenta que nos permite explorar os arquivos e diretórios da nossa estação de trabalho, compilar código, executar programas, e até mesmo instalar novos programas no ambiente de desenvolvimento. E do lado esquerdo, está o "navegador de arquivos", que mostra todos os arquivos e pastas armazenados no seu IDE.

Comece clicando na sua janela do terminal. Você verá que o "prompt" do terminal é parecido com a linha abaixo.

~/ $

Clique dentro da janela do terminal e digite

mkdir ~/problemas1/

seguido de Enter para criar um diretório (também chamado de "pasta") chamado problemas1 no seu diretório inicial. Não se esqueça do espaço entre mkdir and ~/problemas1! Saiba que ~ indica o seu diretório inicial e que ~/problemas1 indica um diretório chamado problemas1 localizado em ~.

A partir de agora, executar um comando significa digitar o comando na janela do terminal e apertar Enter. Comandos são "case-sensitive", o que significa que o comando não funcionará se você digitá-lo em letras maiúsculas, quando deveria digitar em letras minúsculas, e vice-versa.

Agora, execute

cd ~/problemas1/

para que você seja movido para esse diretório (para que você abra o diretório). O prompt da janela do terminal será algo parecido com a linha abaixo

~/problemas1/ $

Caso não seja essa a linha sendo exibida, refaça os passos e veja onde você errou.

Agora, execute

mkdir ~/problemas1/ola

para criar um novo diretório chamado ola dentro do diretório problemas1. Em seguida, execute

cd ~/problemas1/ola

para que você seja movido para esse diretório recentemente criado.

Agora, vamos escrever nosso primeiro programa? No menu superior do IDE, clique na aba File (que significa "arquivo"), clique na opção New File (que significa "novo arquivo"), e salve o novo arquivo (clicando na opção Save, que significa "salvar", da aba File) com o nome ola.c dentro do diretório ~/problemas1/ola. Prossiga digitando exatamente as seguintes linhas dentro do arquivo, para que você possa escrever seu primeiro programa:

#include <stdio.h>

int main(void)
{
    printf("olá, mundo\n");
}

Perceba que o CS50 IDE adiciona "coloração de sintaxe" (destacamento) à medida que você digitar o código. Entretanto, a escolha de cores do CS50 IDE pode ser diferente das mostradas aqui. Essas cores não são salvadas dentro do arquivo, elas são simplesmente adicionadas pelo CS50 IDE para fazer com que alguns comandos e blocos de código importantes fiquem destacados. Caso você não tivesse salvo o arquivo como ola.c, o CS50 IDE não saberia (por conta do nome da extensão do arquivo) que você estava escrevendo código em linguagem C, e nesse caso, as cores estariam ausentes.

Listando Arquivos

Em seguida, na janela do terminal, imediatamente à direita do prompt (~/problemas1/ola/ $), execute

ls

Você está vendo apenas ola.c, correto? Isso é por que você acabou de listar os arquivos dentro da pasta ola. Para ser mais específico, você executou um comando chamado ls, que é abreviação de "lista". É um comando usado com tanta frequência que cientistas da computação decidiram chamar esse comando de ls para economizar alguns pressionamentos de tecla. Faz sentido?

Compilando Programas

Agora, antes de podermos executar o programa ola.c, lembre-se que nós precisamos compilar o código usando um compilador (neste caso, clang), para traduzir o código-fonte do programa em código de máquina (zeros e uns). Execute o comando abaixo para fazer isso:

clang ola.c

E em seguida, execute o comando abaixo novamente:

ls

Dessa vez, você verá não apenas o arquivo ola.c mas também um arquivo chamado a.out, certo? (Você pode ver o mesmo arquivo graficamente se você clicar no ícone de pasta para exibir o "navegador de arquivos".) Isso é por que clang transformou o códifo-fonte do programa em código de máquina, e que foi salvo no arquivo a.out, que por sua vez é a abreviação de "assembler output". Mais informações sobre isso em um outro momento.

Agora, execute o programa com o comando abaixo.

./a.out

Olá, mundo!

Nomeando Programas

Bem, a.out não é o nome mais amigável para um programa. Portanto, vamos compilar ola.c novamente, desta vez salvando o código de máquina em um arquivo chamado ola. Para isso, execute a linha abaixo./p> 

clang -o ola ola.c

Tenha atenção para não esquecer nenhum dos espaços entre as palavras. Em seguida, execute o comando abaixo novamente:

ls

Você verá agora não apenas ola.c (e a.out) mas também um arquivo chamado ola, certo? Isso é por que -o é um argumento de linha de comando, que também são as vezes chamados de flag, que diz para clang gerar um arquivo chamado ola. Execute o comando abaixo para testar o programa com o novo nome.

./ola

Olá, mundo, novamente!

Tornando as Coisas Mais Fáceis

Lembre-se que nós podemos automatizar o processo de executar clang, deixando para o comando make descobrir como compilar o programa, economizando, dessa maneira, nosso tempo. Execute a linha abaixo para compilar este programa uma última vez.

make ola

Você está vendo que make executa clang com vários outros argumentos de linha de comando? Mais detalhes sobre isso também em um outro momento!

Agora, execute o programa, uma última vez, com a linha abaixo.

./ola

Ufa!

Recebendo Entrada de Dados do Usuário

Como você já deve ter percebido, não importa como você compile ou execute este programa, ele só imprime olá, mundo na tela. Vamos personalizar o programa um pouco, assim como fizemos na aula.

Modifique este programa de maneira que primeiro ele peça para o usuário para informar seu nome e em seguida imprime olá Fulano, onde Fulano é o nome que o usuário digitou.

Assim como antes, não se esqueça de compilar seu programa com o comando:

make ola

E não se esqueça de executar seu programa, testando ele com diferentes entradas de dados, usando o comando:

./ola

Dicas

Não se lembra de como pedir ao usuário para informar seu nome?

Lembre-se que você pode usar a função get_string como mostrado na linha abaixo, armazenando o valor de retorno da função get_string em uma variável chamada nome do tipo string. 

string nome = get_string("Qual é seu nome?\n");

Não se lembra de como formatar uma string?

Não se lembra de como juntar (concatenar) o nome do usuário com um cumprimento? Lembre-se que você usar printf não apenas para imprimir mas também para formatar uma string (por isso o f no printf), como mostrado abaixo, onde nome é uma string.

printf("olá, %s\n", nome);

Erro "Use of undeclared identifier"?

Ao tentar compilar o programa, o terminal exibe a seguinte mensagem no topo de uma lista de erros?

error: use of undeclared identifier 'string'; did you mean 'stdin'?

Tradução:

erro: uso de identificador 'string' não declarado; você quis dizer 'stdin'?

Lembre-se que, para usar get_string, você precisa incluir cs50.h (dentro do qual get_string está declarado) no topo do seu arquivo, com a linha:

#include <cs50.h>

Como Testar Seu Código

Execute a linha abaixo para avaliar a correção do seu código usando a ferramenta check50. Não se esqueça de compilar e testar o código você também!

check50 cs50/problems/2020/x/hello

Execute a linha abaixo para avaliar o estilo do seu código usando a ferramenta style50.

style50 ola.c

Como Enviar

Execute a linha abaixo, em seguida informe sua conta GitHub e sua senha quando o terminal pedir. Por medidas de segurança, você verá asteriscos (*) ao invés dos caracteres reais na sua senha.

submit50 cs50/problems/2020/x/hello
